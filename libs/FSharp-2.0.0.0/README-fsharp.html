The F# 2.0 SP1 Free Tools Release

Welcome to the F# 2.0 SP1 Free Tools Release. F# is a typesafe, scalable, succinct, typed, efficient programming language for the .NET platform. The release contains:

  • The F# command-line compiler and library components
  • F# Interactive, for dynamic execution and interactive evaluation
  • The F# components for Visual Studio 2008 and Visual Studio 2010
  • Versions of the FSharp.Core library for Silverlight 2.0, 3.0, 4.0 and Windows Phone 7

Full details can be found at the F# Developer Center or the F# Microsoft Research website . There you can find additional F# learning resources, links to the F# community and more.  You may be interested in tracking Don Syme's F# blog and/or blogging about your own experience using F#. If you are getting started with F#, try the following sites:

  • The F# developer center
  • The F# community site at hubFS
  • The F# samples

Requirements

  • To compile F# programs you need one of:
    • The .NET Framework SDK or .NET Framework Redistributable Components, Versions 2.0, 3.0, 3.5 and 4.0.
    • The freely available Visual Studio 2010 Shell, Integrated Edition.
    • Visual Studio 2010 Professional Edition or above.
    • The freely available Visual Studio 2008 Shell, Integrated Edition.
    • Visual Studio 2008 Standard Edition or above.
    • Another CLI implementation (e.g. Mono). See the notes on using Mono below. Always ensure you have the latest stable Mono release installed (e.g. the Mono 2.10.1 release or later as of the time of writing) - some versions of Linux come with very old Mono releases.
  • If you are not using Visual Studio, you will need an alternate editor. If you use Emacs you will probably want to use the Emacs F# mode.. You may also like to use MonoDevelop., or a command line prompt to run F# Interactive and the command line compiler fsc.exe.

  • Find other F# community resources at http://msdn.microsoft.com/fsharp

Using F# with Mono

You may run F# programs in conjunction with other CLI implementations such as Mono. Please read the notes below. Some more information is available at the F# Wiki.

  • Mono version 2.10.1 or later is required.
  • Install the F# libraries using install-mono.sh. This script contains instructions for setting up your Mono environment for conveniently building and using F# programs.
  • Run the compiler using mono bin/fsc.exe. You will want to make an alias for this.
  • Run F# Interactive compiler using mono bin/fsi.exe. You may wish to make an alias for this.
  • Ahead-of-time compilation can be useful for F# assemblies, but there have been reports of issues with using this on Mono. If you find out that it works on your version of Mono, go ahead and run it on fsc.exe, fsi.exe, FSharp.Compiler.dll, and FSharp.Core.dll to improve startup times for mono bin/fsc.exe.
  • F# has in the past exposed Mono bugs in corner cases. If you see a problem, this is likely to be the result of a bug in Mono. Please be sure to investigate and report these to both the F# team and through the Mono bug reporting process.
